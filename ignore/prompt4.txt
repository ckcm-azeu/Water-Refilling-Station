## PROJECT CONTEXT RECAP

Azeu Water Station is a web-based order and delivery management system for a water refilling station in the Philippines. It has 4 roles: Customer, Rider, Staff, Admin (with Super Admin). The tech stack is pure PHP (no frameworks), MySQL on XAMPP (database name: `station_azeu`), and hand-written HTML/CSS/JS. No frameworks, no Bootstrap, no Tailwind.

**Staff** are operational managers who:
- Cannot register themselves — accounts are created by Admin/Super Admin only
- Are always `status='active'` from creation (skip pending approval)
- Approve or delete pending customer accounts after registration
- Create Customer and Rider accounts (NOT Staff or Admin — only Admin can create Staff)
- Edit and delete (soft-delete) Customer and Rider accounts
- Confirm customer orders (which triggers stock decrement from inventory)
- Assign riders to confirmed orders (manually select or auto-assign least-loaded available rider)
- Reassign riders on assigned orders
- Update order status at any stage (Pending, Confirmed, Assigned, On Delivery, Delivered, Ready for Pickup, Picked Up)
- Cancel orders at any status with a mandatory reason
- Add staff comments to orders (visible to customers, e.g., "Waiting for available rider")
- Mark pickup orders as "Ready for Pickup" and "Picked Up"
- View inventory in read-only mode (see stock counts, prices, statuses — cannot add/edit/remove items)
- View rider performance statistics (completed deliveries, success rates, current loads)
- Review and approve/deny customer cancellation appeals
- View all order history with filters and search
- Manage their own profile and password
- Perform bulk actions on orders (bulk confirm, bulk cancel, bulk assign)

**Staff CANNOT:**
- Create Staff, Admin, or Super Admin accounts (only Admin+ can do this)
- See Staff, Admin, or Super Admin accounts in Account Management (only Customer and Rider accounts visible)
- Modify inventory (read-only access)
- Access Analytics, Session Logs, or System Settings pages (Admin-only features)
- Modify or delete the Super Admin account

## ORDER LIFECYCLE (Staff Perspective — Full Control)

Staff has the highest operational authority over orders (same level as Admin for order management). Staff can perform all status transitions except they cannot modify system settings.

### Delivery Orders (Cash on Delivery) — Full Flow
```
PENDING ──► CONFIRMED ──► ASSIGNED ──► ON DELIVERY ──► DELIVERED ──► ACCEPTED (by customer)
   │             │             │              │               │
   │             │             │              │               └── Final (no staff action)
   │             │             │              └── Staff can: Update status, Cancel
   │             │             └── Staff can: Reassign rider, Cancel
   │             └── Staff can: Assign rider, Cancel, Mark Ready for Pickup (if pickup type)
   └── Staff can: Confirm (triggers stock decrement), Cancel
```

### Pickup Orders — Full Flow
```
PENDING ──► CONFIRMED ──► READY FOR PICKUP ──► PICKED UP ──► ACCEPTED (by customer)
   │             │                │                 │
   │             │                │                 └── Final
   │             │                └── Staff can: Mark Picked Up, Cancel
   │             └── Staff can: Mark Ready for Pickup, Cancel
   └── Staff can: Confirm, Cancel
```

### Staff Order Actions by Status

| Current Status | Available Actions | Details |
|----------------|-------------------|---------|
| **Pending** | Confirm, Cancel | Confirm: decrements stock, creates notification. Optionally assign rider immediately ("Assign now?" dialog). Cancel: reason required, stock NOT restored (never decremented). |
| **Confirmed** (no rider) | Assign Rider, Ready for Pickup (pickup only), Cancel | Assign: Select rider manually or Auto-assign. Cancel: reason required, stock IS restored. |
| **Confirmed** (pickup, no rider) | Ready for Pickup, Cancel | Move to Ready for Pickup. |
| **Assigned** (has rider) | Reassign Rider, Cancel | Reassign: select different rider. Cancel: reason required, stock IS restored, rider notified. |
| **On Delivery** | Update Status, Cancel | Update: change to Pending/Confirmed/Delivered/any status. Cancel: reason required, stock IS restored, rider notified. |
| **Ready for Pickup** | Picked Up, Cancel | Mark Picked Up: income counted. Cancel: stock restored. |
| **Delivered** | (View only) | Cannot change. Income already counted. |
| **Picked Up** | (View only) | Cannot change. Income already counted. |
| **Accepted** | (View only) | Customer confirmed receipt. |
| **Cancelled** | (View only in history) | Final state. |

### Stock Management Rules (Staff Must Understand)
- **Stock decremented** when order moves from `Pending` → `Confirmed`
- **Stock restored** when any order that was `Confirmed` or beyond is `Cancelled`
- **Stock NOT restored** when a `Pending` order is cancelled (stock was never decremented)
- Staff can see current stock levels in the read-only Inventory page to make informed decisions

### Auto-Assign Rider Algorithm
When staff clicks "Auto" assign:
1. Get all riders WHERE `role='rider'` AND `status='active'` AND `is_available=1` AND `deleted_at IS NULL`
2. For each rider, COUNT their current orders WHERE status IN `('assigned','on_delivery')` → this is "current_load"
3. Sort by `current_load ASC` (least loaded first)
4. If tied, sort by `rider.id ASC` (first registered)
5. If NO available riders → return failure, order stays Confirmed, staff can optionally add a comment like "Waiting for available rider" visible to the customer
6. If rider found → assign, set status to `assigned`, create `delivery_priority` entry, notify rider + customer

### Order Grouping in Manage Orders
Orders from the **same customer** with the **same status** should be visually grouped in the table:
- They appear as one expandable/collapsible row showing the customer name, total items count across all orders, and combined status
- Clicking the row expands to show individual orders underneath
- Bulk actions on the grouped row apply to ALL orders in that group
- This makes it easier for staff to manage multiple orders from one customer

**PHP Logic (top of file):**
```
1. Include auth_check.php, require_role([ROLE_STAFF])
2. Fetch dashboard stats:
   - Pending orders count: SELECT COUNT(*) FROM orders WHERE status='pending'
   - Confirmed orders count: SELECT COUNT(*) FROM orders WHERE status='confirmed'
   - On delivery orders count: SELECT COUNT(*) FROM orders WHERE status='on_delivery'
   - Pending accounts count: SELECT COUNT(*) FROM users WHERE status='pending'
   - Total riders count: SELECT COUNT(*) FROM users WHERE role='rider' AND status='active' AND deleted_at IS NULL
   - Available riders count: SELECT COUNT(*) FROM users WHERE role='rider' AND status='active' AND is_available=1 AND deleted_at IS NULL
3. Fetch pending accounts preview (max 3, newest first):
   SELECT * FROM users WHERE status='pending'
   AND role IN ('customer','rider')
   ORDER BY created_at DESC LIMIT 3
4. Fetch pending orders preview (max 3, newest first):
   SELECT o.*, u.full_name as customer_name, u.phone as customer_phone,
   GROUP_CONCAT(oi.item_name) as item_names
   FROM orders o
   LEFT JOIN users u ON o.customer_id = u.id
   LEFT JOIN order_items oi ON o.id = oi.order_id

   GROUP BY o.id
   ORDER BY o.created_at DESC LIMIT 3
5. Include header.php, sidebar.php
```

```javascript
    let filterRole = '';

    if (['customer', 'rider'].includes(currentFilter)) {

      filterStatus = currentFilter;
    }

    const params = {
      filter_role: filterRole,
      filter_status: filterStatus,
      search: searchText,
      page: currentPage
    };

    document.getElementById('accountsBody').innerHTML =
      '<tr class="loading-row"><td colspan="10"><div class="spinner"></div></td></tr>';

      .then(data => {
        if (!data.accounts || data.accounts.length === 0) {
          document.getElementById('accountsBody').innerHTML = '';
          document.getElementById('emptyAccounts').style.display = 'flex';
          document.getElementById('accountsPagination').innerHTML = '';
          return;
        }
        document.getElementById('emptyAccounts').style.display = 'none';
        renderAccounts(data.accounts);
        renderPagination(data.total, data.page, data.per_page, 'accountsPagination', loadAccounts);
      })
      .catch(() => showToast('Failed to load accounts', 'error'));

  // ===== RENDER ACCOUNTS =====

            <span class="material-icons">edit</span>

          <button class="btn-icon btn-danger" title="Delete" onclick="deleteAccount(${acc.id}, '${escapeHtml(acc.username)}')">
            <span class="material-icons">delete</span>
          </button>
        `;
      }

      // Get default address for display

    return new Date(d).toLocaleDateString('en-US', { month:'short', day:'numeric', year:'numeric' })

    const container = document.getElementById(containerId);
    container.innerHTML = '';
    if (totalPages <= 1) return;

    const prev = document.createElement('button');
    prev.className = 'page-btn'; prev.textContent = 'Previous'; prev.disabled = page <= 1;
    prev.addEventListener('click', () => { currentPage--; loadFunc(); });
    container.appendChild(prev);

    for (let i = Math.max(1,page-2); i <= Math.min(totalPages,page+2); i++) {
      const pb = document.createElement('button');
      pb.className = `page-btn ${i===page?'active':''}`;
      pb.textContent = i;
      pb.addEventListener('click', () => { currentPage = i; loadFunc(); });
      container.appendChild(pb);
    }

    const next = document.createElement('button');
    next.className = 'page-btn'; next.textContent = 'Next'; next.disabled = page >= totalPages;
    next.addEventListener('click', () => { currentPage++; loadFunc(); });
    container.appendChild(next);

    const info = document.createElement('span');
    info.className = 'pagination-info';
    info.textContent = `Showing ${(page-1)*perPage+1}–${Math.min(page*perPage,total)} of ${total}`;
    container.appendChild(info);
  }

});
```

### `staff/js/manage_orders.js`

```javascript
document.addEventListener('DOMContentLoaded', function() {

  let currentFilter = 'all';
  let currentPage = 1;

  const perPage = 50;

  // ===== SEARCH =====

    clearTimeout(searchTimeout);

  // ===== LOAD ORDERS =====
  function loadOrders() {
    const activeStatuses = 'pending,confirmed,assigned,on_delivery,ready_for_pickup';
    const statusParam = currentFilter === 'all' ? activeStatuses : currentFilter;

    const params = {
      status: statusParam,
      search: searchText,
      page: currentPage,
      scope: 'manage'
    };

    document.getElementById('ordersBody').innerHTML =
      '<tr class="loading-row"><td colspan="13"><div class="spinner"></div></td></tr>';

        }

        renderPagination(data.total, data.page, data.per_page);

          const highlightRow = document.querySelector(`tr[data-order-id="${highlightOrderId}"]`);
          if (highlightRow) {
            highlightRow.classList.add('highlight');
            highlightRow.scrollIntoView({ behavior: 'smooth', block: 'center' });
          }

      })
      .catch(() => showToast('Failed to load orders', 'error'));

  // ===== RENDER ORDERS =====
  function renderOrders(orders) {
    const tbody = document.getElementById('ordersBody');
    tbody.innerHTML = '';

    const paymentLabels = { cod: 'Cash on Delivery', pickup: 'Pickup at Store', online: 'Online' };
    const typeLabels = { delivery: 'Delivery', pickup: 'Pickup' };

    // Group orders by customer + status for collapsing
    const groups = {};
    orders.forEach(order => {
      const key = `${order.customer_id}_${order.status}`;
      if (!groups[key]) groups[key] = [];
      groups[key].push(order);
    });

    Object.values(groups).forEach(group => {
      if (group.length > 1) {
        // Render group header (collapsible)
        renderGroupHeader(tbody, group);
        // Render individual orders (hidden initially)
        group.forEach(order => {
          renderOrderRow(tbody, order, paymentLabels, typeLabels, true); // isGroupChild = true
        });
      } else {
        // Single order — render normally
        renderOrderRow(tbody, group[0], paymentLabels, typeLabels, false);
      }
    });

    initCheckboxListeners();
  }

  function renderGroupHeader(tbody, group) {
    const first = group[0];
    const totalItems = group.reduce((sum, o) => sum + o.items.reduce((s, i) => s + parseInt(i.quantity), 0), 0);
    const groupId = `group_${first.customer_id}_${first.status}`;

    const tr = document.createElement('tr');
    tr.className = 'order-group-header';
    tr.dataset.groupId = groupId;

      </td>
      <td>${totalItems} items total</td>
      <td colspan="3"></td>
      <td colspan="2"></td>
      <td><span class="badge badge-${first.status.replace(/_/g,'-')}">${formatStatus(first.status)}</span></td>
      <td></td>
      <td class="actions-cell">
        ${getGroupActions(first.status, group.map(o => o.id))}
      </td>
    `;

      const children = tbody.querySelectorAll(`.group-child[data-group-id="${groupId}"]`);
      const isExpanded = this.textContent === 'expand_less';
      children.forEach(c => c.style.display = isExpanded ? 'none' : '');
      this.textContent = isExpanded ? 'expand_more' : 'expand_less';
    });

    tbody.appendChild(tr);

    const prev = document.createElement('button'); prev.className='page-btn'; prev.textContent='Previous'; prev.disabled=page<=1;
    prev.addEventListener('click', () => { currentPage--; loadOrders(); }); container.appendChild(prev);
    for (let i=Math.max(1,page-2); i<=Math.min(totalPages,page+2); i++) {

    next.addEventListener('click', () => { currentPage++; loadOrders(); }); container.appendChild(next);
    const info = document.createElement('span'); info.className='pagination-info';
    info.textContent = `Showing ${(page-1)*perPage+1}–${Math.min(page*perPage,total)} of ${total}`;
    container.appendChild(info);
```

```javascript
document.addEventListener('DOMContentLoaded', function() {

  loadRiderStats();

  function loadRiderStats() {
    ajaxGet('/api/riders/get_statistics.php')
      .then(data => {
        document.getElementById('totalRiders').textContent = data.total_riders || 0;
        document.getElementById('availableRiders').textContent = data.available_riders || 0;
        renderRiders(data.riders || []);
      })
      .catch(() => showToast('Failed to load rider statistics', 'error'));
  }

  function renderRiders(riders) {
    const tbody = document.getElementById('ridersBody');
    tbody.innerHTML = '';

    if (riders.length === 0) {
      tbody.innerHTML = '<tr><td colspan="8" class="text-center text-muted">No riders found</td></tr>';
      return;
    }

    riders.forEach((rider, index) => {
      const rank = index + 1;
      const rankClass = rank <= 3 ? `rank-${rank}` : 'rank-default';

      // Determine status display
      let statusHtml = '';
      if (rider.on_delivery_count > 0) {
        statusHtml = '<span class="badge badge-on-delivery">On Delivery</span>';
      } else if (rider.is_available) {
        statusHtml = '<span class="badge badge-confirmed">Available</span>';
      } else {
        statusHtml = '<span class="badge badge-cancelled">Unavailable</span>';
      }

      const successRate = rider.completed > 0
        ? ((rider.completed / (rider.completed + rider.cancelled)) * 100).toFixed(1)
        : '0.0';

      const tr = document.createElement('tr');
      tr.innerHTML = `
        <td><span class="rank-number ${rankClass}">${rank}</span></td>
        <td><strong>${escapeHtml(rider.full_name)}</strong></td>
        <td>${escapeHtml(rider.email)}</td>
        <td>${statusHtml}</td>
        <td><strong class="text-success">${rider.completed}</strong></td>
        <td>${rider.pending > 0 ? `<strong class="text-warning">${rider.pending}</strong>` : '0'}</td>
        <td>
          <div class="success-rate-cell">
            <div class="success-rate-bar" style="width:${successRate}%"></div>
            <span>${successRate}%</span>
          </div>
        </td>
        <td>
          <button class="btn-icon btn-info" title="Show Full Info" onclick="showRiderInfo(${rider.rider_id})">
            <span class="material-icons">info</span>
          </button>
        </td>
      `;
      tbody.appendChild(tr);
    });
  }

  // Show Rider Full Info
  window.showRiderInfo = function(riderId) {
    ajaxGet('/api/accounts/get_accounts.php', { user_id: riderId })
      .then(data => {
        const r = data.account || data.accounts?.find(a => a.id === riderId);
        if (!r) { showToast('Rider not found', 'error'); return; }

        Swal.fire({
          title: 'Rider Information',

              <p><strong>Full Name:</strong> ${escapeHtml(r.full_name)}</p>

              <p><strong>Phone:</strong> ${escapeHtml(r.phone)}</p>
              <p><strong>Status:</strong> <span class="badge badge-${r.status}">${r.status}</span></p>
              <p><strong>Available:</strong> ${r.is_available ? 'Yes' : 'No'}</p>
              <p><strong>Member Since:</strong> ${formatDate(r.created_at)}</p>

          `,

          confirmButtonColor: 'var(--primary)'
        });
      }).catch(() => showToast('Failed to load rider info', 'error'));
  };

  function escapeHtml(t) { const d=document.createElement('div'); d.textContent=t||''; return d.innerHTML; }
  function formatDate(d) { if(!d) return '—'; return new Date(d).toLocaleDateString('en-US',{month:'short',day:'numeric',year:'numeric'}); }
```

## IMPORTANT IMPLEMENTATION NOTES FOR STAFF PAGES

1. **Staff CANNOT see Staff/Admin/Super Admin accounts** in Account Management. The `api/accounts/get_accounts.php` endpoint filters by role when the caller is `staff` — only returning `customer` and `rider` accounts. The frontend should never attempt to display or modify higher-role accounts.

2. **Staff CANNOT create Staff/Admin accounts.** The "Add Account" dialog only offers `Customer` and `Rider` as role options. The API enforces this restriction.

3. **Order grouping in Manage Orders:** When multiple orders exist from the same customer with the same status, they should be grouped into a collapsible row. The group header shows customer name, order count, and combined actions. Clicking the expand arrow reveals individual orders. Individual orders within a group are indented slightly. Actions on the group header apply to ALL orders in the group.

4. **Rider assignment dialog** shows a list of available riders with their current delivery load (number of assigned + on_delivery orders). Staff can select a specific rider or click "Auto-Assign" which selects the least-loaded available rider. If no riders are available, staff is offered the option to add a comment to the order explaining the delay.

5. **Staff can cancel orders at ANY status** but must always provide a reason. The cancellation record includes who cancelled it (staff's user_id). Stock is restored automatically for orders that were already confirmed.

6. **Inventory page is READ-ONLY.** No add, edit, restock, or delete buttons. The table shows current stock levels, prices, and status. Items below the `low_stock_threshold` setting are highlighted with a warning icon. This helps staff make informed decisions when confirming orders.

7. **Rider Statistics page** shows all riders ranked by completed deliveries. Each rider shows: name, email, current status (Available/Unavailable/On Delivery), completed count, pending count, and success rate. The "Show Info" button shows a dialog with full rider details (no password).

8. **Cancellation Appeals** are shown in the Account Management page below the accounts table. Staff can approve (unflag customer, reset cancellation count) or deny (keep flagged, add admin notes) appeals.

9. **All tables render max 50 items per page** with pagination.

10. **Every AJAX call** uses `ajaxPost()`/`ajaxGet()` from `global.js` with automatic CSRF token attachment and `.catch()` error handling.

11. **Dark mode support** via CSS variables only — never hardcode colors in page-specific CSS.

12. **All date/time** formatted for Philippines Manila timezone.

13. **Glass card effect** used for: stat cards, preview sections, settings sections, rider statistics table wrapper. Subtle, not overused.

14. **Staff settings page** is simple: Profile (cannot change username) + Password change. No system settings, no addresses, no appeals, no availability.

15. **The `rider-select-item` class** used in the assign rider dialog needs CSS styling:
