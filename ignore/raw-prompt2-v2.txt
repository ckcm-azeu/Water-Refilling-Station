## 1. CONFIRMED ORDER STATUS STATE MACHINES

### Delivery Orders (COD)
```
                                            ┌─── Cancelled
                                            │    (by Customer)
                                            │
PENDING ──► CONFIRMED ──► ASSIGNED ──► ON DELIVERY ──► DELIVERED ──► ACCEPTED
   │              │            │                                     (optional,
   │              │            │                                      by Customer)
   │              │            └─── Cancelled (by Staff/Admin)
   │              └─── Cancelled (by Staff/Admin)
   └─── Cancelled (by Customer or Staff/Admin)
```

**Rules:**
- Customer can cancel at: `Pending` only
- Staff/Admin can cancel at: ANY stage (with reason)
- Rider marks: `On Delivery` → `Delivered`
- Expected Delivery Date is auto-set when rider marks `On Delivery`
- `Delivered` → income is counted
- `Accepted` is optional, only by customer, purely a receipt acknowledgment
- Stock decremented at `Confirmed`, restored at `Cancelled`

### Pickup Orders (Pay at Store)
```
PENDING ──► CONFIRMED ──► READY FOR PICKUP ──► PICKED UP
   │              │
   │              └─── Cancelled (by Staff/Admin)
   └─── Cancelled (by Customer or Staff/Admin)
```

**Rules:**
- No rider assigned, delivery fee = ₱0
- Staff/Admin moves from `Confirmed` → `Ready for Pickup`
- Staff/Admin marks `Picked Up` when customer collects
- `Picked Up` → income is counted
- Customer can optionally mark as `Accepted` after `Picked Up`

## 6. PAGE-BY-PAGE DETAILED FLOW

### PUBLIC PAGES

#### `index.php` — Login
```
LOAD → Check session
├── If logged in → Redirect to /{role}/dashboard.php
└── If not logged in → Show login form
    ├── Fields: Username, Password
    ├── Link: "Forgot Password?" → forgot_password.php
    ├── Link: "Create Account" → register.php
    └── ON SUBMIT (AJAX → api/auth/login.php):
        ├── Check maintenance mode → if ON and role != admin/super_admin → show maintenance message
        ├── Check login attempts → if >= max → show "Account locked for X minutes"
        ├── Validate credentials (decrypt/compare)
        ├── Check account status:
        │   ├── pending → Login allowed but show restricted dashboard
        │   ├── active → Full login
        │   ├── flagged → Login allowed but ordering disabled, show appeal option
        │   └── deleted → "Account not found"
        ├── Create session (user_id, role, username, full_name)
        ├── Load user_preferences → apply dark_mode
        ├── Check force_dark_mode setting → override if ON
        ├── Log to session_logs (login, IP)
        └── Redirect to /{role}/dashboard.php
```

#### `register.php` — Customer Registration
```
LOAD → Check session
├── If logged in → Redirect to dashboard
└── Show registration form
    ├── Fields: Full Name, Username, Email, Full Address, Phone, Password, Confirm Password
    ├── Floating animated labels on all fields
    ├── Real-time validation (username uniqueness via AJAX)
    └── ON SUBMIT (AJAX → api/auth/register.php):
        ├── Server-side validation (all fields, email format, username unique, password strength)
        ├── Create user (role=customer, status=pending)
        ├── Create default address in customer_addresses (is_default=1)
        ├── Create user_preferences row
        ├── Show success dialog: "Account created! Awaiting approval."
        └── Redirect to login
```

#### `forgot_password.php`
```
Show email input form → SUBMIT → api/auth/forgot_password.php
├── Check email exists AND role=customer AND status=active
├── Generate random token, store in password_resets (expires +1 hour)
├── Send email via PHPMailer with reset link
└── Show: "If this email exists, a reset link has been sent."
```

#### `reset_password.php?token=xxx`
```
LOAD → Validate token (exists, not expired, not used)
├── Invalid → Show error, link to forgot_password
└── Valid → Show new password form
    └── ON SUBMIT → api/auth/reset_password.php
        ├── Update user password (encrypt with AES)
        ├── Mark token as used
        └── Show success, redirect to login
```

#### `receipt.php?token=xxx`
```
LOAD → Query orders WHERE receipt_token = token
├── Not found → 404
└── Found → Render receipt:
    ├── Station name + logo
    ├── Receipt date, Order ID, Order date
    ├── Customer name, address, phone
    ├── Items table: Name, Qty, Unit Price, Subtotal
    ├── Subtotal, Delivery Fee, Total Amount
    ├── Payment type
    ├── QR code (encodes this receipt URL)
    ├── Button: "Download as PDF" (html2pdf.js)
    ├── Button: "Download as Image" (html2canvas)
    └── Button: "Go to Dashboard" (if logged in) or "Go to Login"
```

## 8. NOTIFICATION SYSTEM DESIGN

### Two Types

**Type 1 — SweetAlert2 Dialogs (Immediate Actions)**
- Confirmation prompts ("Are you sure?")
- Success/Error messages after actions
- Form dialogs (edit, cancel reason, etc.)
- Triggered by user action, blocking

**Type 2 — Bell Notifications (Persistent)**
- Stored in `notifications` table
- Bell icon in header with unread count badge
- Click bell → dropdown showing recent 10 notifications
- Each notification: icon + title + time ago + link
- [Mark All Read] button
- Refreshed on page load (no polling needed since refresh-based)

### When Notifications Are Created

| Event | Notified User(s) | Title |
|-------|------------------|-------|
| New order placed | All Staff + Admin | "New order #{id} from {customer}" |

| Order On Delivery | Customer | "Order #{id} is on the way!" |

## 9. KEY ALGORITHMS

### Auto-Assign Rider Algorithm
```
FUNCTION autoAssignRider(order_id):
    1. Get all riders WHERE:
       - role = 'rider'
       - status = 'active'
       - is_available = 1
       - NOT deleted
    2. For each rider, COUNT orders WHERE:
       - rider_id = this rider
       - status IN ('assigned', 'on_delivery')
       → This is their "current_load"
    3. Filter out riders where ALL their orders are on_delivery
       (they're fully occupied on the road)
       → Actually, INCLUDE them but prefer riders with 0 on_delivery first
    4. Sort by current_load ASC (least loaded first)
    5. If tie, sort by rider.id ASC (first created)
    6. If NO available riders → return NULL (order stays Confirmed with comment)
    7. Assign top rider → update order.rider_id, status='assigned'
    8. Create notification for rider
    RETURN assigned_rider_id or NULL
```

### Cancellation Monthly Reset
```
CRON JOB (or check on each login / page load):
    For each user WHERE role='customer':
        If cancellation_reset_date <= TODAY:
            SET cancellation_count = 0
            SET cancellation_reset_date = FIRST_DAY_OF_NEXT_MONTH
            If status = 'flagged' AND no pending appeals:
                Keep flagged (requires appeal to unflag)
```
> Since you're using XAMPP (no cron), run this check in `config/session.php` on every authenticated page load. It only runs a quick query so no performance impact.

### Pending Account Expiry Cleanup
```
Same approach — run on page load in session.php:
    DELETE FROM users
    WHERE status = 'pending'
    AND DATEDIFF(NOW(), created_at) > {pending_expiry_days setting}
```

### Password Encryption Toggle Migration
```
WHEN admin toggles "Encrypt DB Passwords":
    If turning ON:
        SELECT all users
        For each: UPDATE password = encrypt(password, KEY)
    If turning OFF:
        SELECT all users
        For each: UPDATE password = decrypt(password, KEY)
    Save setting
```

### Stock Validation at Checkout
```
FUNCTION validateStock(cart_items[]):
    removed_items = []
    For each item in cart_items:
        current_stock = SELECT stock_count FROM inventory WHERE id = item.inventory_id
        If current_stock < item.quantity:
            If current_stock == 0:
                removed_items.push(item) // completely unavailable
            Else:
                item.quantity = current_stock // reduce to available
                item.adjusted = true
    RETURN { valid_items, removed_items, adjusted_items }
```
