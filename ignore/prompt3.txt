## PROJECT CONTEXT RECAP

Azeu Water Station is a web-based order and delivery management system for a water refilling station in the Philippines. It has 4 roles: Customer, Rider, Staff, Admin (with Super Admin). The tech stack is pure PHP (no frameworks), MySQL on XAMPP (database name: `station_azeu`), and hand-written HTML/CSS/JS. No frameworks, no Bootstrap, no Tailwind.

**Riders** are delivery personnel who:
- Cannot register themselves — accounts are created by Staff or Admin only
- Are always `status='active'` from creation (skip pending approval since created by authority)
- View deliveries assigned to them by Staff/Admin
- Organize their delivery priority order via drag-and-drop
- Set themselves as Available or Unavailable (but cannot go Unavailable while having `on_delivery` orders)
- Mark orders as `On Delivery` (which auto-sets `expected_delivery_date` to today)
- Mark orders as `Delivered` (which sets `delivered_at` timestamp and counts to store income)
- Can cancel assigned deliveries with a reason (order reverts to `Confirmed` status with `rider_id` set to NULL for Staff to reassign)
- View their delivery history (completed and cancelled)
- Manage their own profile and password in Settings

## ORDER LIFECYCLE (Rider Perspective)

### What the Rider Sees

Riders only see orders that are **assigned to them** (`rider_id = their user ID`). They do NOT see pending or confirmed orders — those are managed by Staff/Admin. Once Staff/Admin assigns a rider to an order, it appears in the rider's "Assigned Deliveries" page.

### Status Transitions a Rider Can Make

```
ASSIGNED ──► ON DELIVERY ──► DELIVERED
    │              │
    │              └── Cancel (with reason) → reverts to CONFIRMED, rider_id = NULL
    └── Cancel (with reason) → reverts to CONFIRMED, rider_id = NULL
```

**Rider actions:**
1. **Mark as On Delivery** — Changes status from `assigned` to `on_delivery`. This automatically sets `expected_delivery_date` to today's date. This means the rider has picked up the items from the station and is on the way to the customer. **Once a rider marks ANY order as On Delivery, they CANNOT set themselves as Unavailable until ALL their on_delivery orders are completed (delivered) or cancelled.**

2. **Mark as Delivered** — Changes status from `on_delivery` to `delivered`. This sets `delivered_at` to NOW(). The order amount is now counted toward store income. A notification is sent to the customer: "Order #{id} has been delivered!"

3. **Cancel Delivery** — Rider can cancel at `assigned` or `on_delivery` status. A reason is REQUIRED. When a rider cancels:
   - The order status reverts to `confirmed` (NOT cancelled — the order still exists)
   - `rider_id` is set to NULL (unassigned)
   - Staff/Admin can then reassign to another rider
   - If the order was `on_delivery`, stock is NOT restored (stock was decremented at confirmation, and the order is still going to be fulfilled — just by a different rider)
   - A notification is sent to Staff/Admin: "Rider {name} cancelled delivery of Order #{id}. Reason: {reason}"
   - This is treated as a rider reassignment request, NOT as an order cancellation

### Delivery Priority System

Riders can reorder their assigned deliveries to plan their route. This is a drag-and-drop feature:

- Each assigned/on_delivery order has a priority number stored in `delivery_priority` table
- Lower `priority_order` number = deliver first (higher priority)
- Default priority is set when the order is assigned (based on assignment order — first assigned = priority 1, etc.)
- Rider can drag-and-drop to reorder on the `assigned_deliveries.php` page
- On reorder, JS sends the new priority list to `api/riders/update_priority.php`
- The table/list displays orders sorted by `priority_order ASC`
- Only `assigned` and `on_delivery` orders are in the priority list (not delivered/cancelled)

### Rider Availability System

- `is_available` column in `users` table (TINYINT: 0 or 1)
- Default: 1 (available) when account is created
- Rider can toggle via a switch on their dashboard
- **Restriction:** Cannot set to Unavailable (0) if they have ANY orders with status `on_delivery`. They must complete or cancel all on_delivery orders first. The API endpoint enforces this — returns an error if rider tries to go unavailable while having on_delivery orders.
- When `is_available = 0`: The rider will NOT be selected by the auto-assign algorithm. Staff/Admin CAN still manually assign orders to unavailable riders (but will see a warning).
- The availability status is shown on the rider's dashboard and in Staff/Admin rider statistics pages.

### 1. `rider/dashboard.php`

**PHP Logic (top of file):**
```
1. Include auth_check.php, require_role([ROLE_RIDER])
2. Get current user data from session
3. Fetch rider's current availability status from DB (fresh read, not session cache):
   SELECT is_available FROM users WHERE id = session.user_id
4. Fetch dashboard stats:
   - Assigned deliveries count:
     SELECT COUNT(*) FROM orders WHERE rider_id = user_id AND status IN ('assigned', 'on_delivery')
   - On delivery count:
     SELECT COUNT(*) FROM orders WHERE rider_id = user_id AND status = 'on_delivery'
   - Completed today count:
     SELECT COUNT(*) FROM orders WHERE rider_id = user_id AND status = 'delivered'
     AND DATE(delivered_at) = CURDATE()
5. Check if rider can toggle availability:
   $has_on_delivery = (on_delivery_count > 0)
   → If true, the toggle switch should be locked to "Available" and cannot be turned off
6. Fetch delivery preview (max 3, sorted by priority):
   SELECT o.*, GROUP_CONCAT(oi.item_name) as item_names,
   u.full_name as customer_name, u.phone as customer_phone,
   ca.full_address as customer_address,
   dp.priority_order
   FROM orders o
   LEFT JOIN order_items oi ON o.id = oi.order_id
   LEFT JOIN users u ON o.customer_id = u.id
   LEFT JOIN customer_addresses ca ON (ca.customer_id = o.customer_id AND ca.is_default = 1)
   LEFT JOIN delivery_priority dp ON (dp.order_id = o.id AND dp.rider_id = o.rider_id)
   WHERE o.rider_id = session.user_id AND o.status IN ('assigned', 'on_delivery')
   GROUP BY o.id
   ORDER BY dp.priority_order ASC, o.created_at ASC
   LIMIT 3
7. Include header.php, sidebar.php
```

### `rider/js/dashboard.js`

```javascript
document.addEventListener('DOMContentLoaded', function() {

  // ===== AVAILABILITY TOGGLE =====
  const toggleInput = document.getElementById('availabilityToggle');
  if (toggleInput && !toggleInput.disabled) {
    toggleInput.addEventListener('change', function() {
      const newState = this.checked; // true = available, false = unavailable
      const confirmMsg = newState
        ? 'Set yourself as Available? You may receive new delivery assignments.'
        : 'Set yourself as Unavailable? You will not receive new delivery assignments.';

      Swal.fire({
        title: newState ? 'Go Available?' : 'Go Unavailable?',
        text: confirmMsg,
        icon: 'question',
        showCancelButton: true,
        confirmButtonText: newState ? 'Yes, Go Available' : 'Yes, Go Unavailable',
        cancelButtonText: 'Cancel',
        confirmButtonColor: newState ? 'var(--success)' : 'var(--warning)',
      }).then(result => {
        if (result.isConfirmed) {
          ajaxPost('/api/riders/toggle_availability.php', {})
            .then(data => {
              if (data.success) {
                showToast(
                  newState ? 'You are now available for deliveries' : 'You are now unavailable',
                  'success'
                );
                // Update visual state
                updateAvailabilityUI(newState);
              } else {
                // Revert toggle
                toggleInput.checked = !newState;
                showToast(data.message || 'Failed to update availability', 'error');
              }
            })
            .catch(err => {
              toggleInput.checked = !newState;
              showToast('An error occurred. Please try again.', 'error');
            });
        } else {
          // User cancelled the dialog — revert the toggle
          toggleInput.checked = !newState;
        }
      });
    });
  }

  function updateAvailabilityUI(isAvailable) {
    const icon = document.querySelector('.availability-icon');
    const label = document.querySelector('.availability-label');
```

```javascript
document.addEventListener('DOMContentLoaded', function() {

  let currentFilter = 'all';
  let currentPage = 1;
  const perPage = 50;
  let sortableInstance = null;

  // Load deliveries on page load
  loadDeliveries();

  // ===== FILTER BUTTONS =====
  document.querySelectorAll('#deliveryFilters .filter-btn').forEach(btn => {
    btn.addEventListener('click', function() {
      document.querySelectorAll('#deliveryFilters .filter-btn').forEach(b => b.classList.remove('active'));
      this.classList.add('active');
      currentFilter = this.dataset.filter;
      currentPage = 1;
      loadDeliveries();
    });
  });

  // ===== LOAD DELIVERIES =====
  function loadDeliveries() {
    const params = {
      status: currentFilter === 'all' ? 'assigned,on_delivery' : currentFilter,
      page: currentPage,
      scope: 'active' // Tell API to return only assigned + on_delivery for rider
    };

    document.getElementById('deliveriesBody').innerHTML =
      '<tr class="loading-row"><td colspan="13"><div class="spinner"></div></td></tr>';
    document.getElementById('emptyDeliveries').style.display = 'none';

    ajaxGet('/api/orders/get_orders.php', params)
      .then(data => {
        if (!data.orders || data.orders.length === 0) {
          document.getElementById('deliveriesBody').innerHTML = '';
          document.getElementById('emptyDeliveries').style.display = 'flex';
          document.getElementById('deliveriesPagination').innerHTML = '';
          destroySortable();
          return;
        }

        document.getElementById('emptyDeliveries').style.display = 'none';
        renderDeliveries(data.orders);
        renderPagination(data.total, data.page, data.per_page);
        initSortable();
      })
      .catch(err => {
        showToast('Failed to load deliveries', 'error');
      });
  }

  // ===== RENDER DELIVERIES TABLE =====
  function renderDeliveries(orders) {
    const tbody = document.getElementById('deliveriesBody');
    tbody.innerHTML = '';

    const paymentLabels = { cod: 'Cash on Delivery', pickup: 'Pickup at Store', online: 'Online' };

    orders.forEach((order, index) => {
      const tr = document.createElement('tr');
      tr.dataset.orderId = order.id;
      tr.dataset.status = order.status;

      // Items display
      const itemsHtml = order.items.map(item =>
        `<span class="item-with-icon">
          <img src="${item.item_icon || '/assets/images/default-item.png'}" class="item-icon-sm">
          ${item.item_name}
        </span>`
      ).join('<br>');

      // Total quantity
      const totalQty = order.items.reduce((sum, i) => sum + parseInt(i.quantity), 0);

      // Customer details formatted:
      // "Customer Name"
      // "📞 Phone Number"
      // "📍 Full Address"
      const customerDetails = `
        <strong>${order.customer_name || 'Unknown'}</strong>
        <span class="customer-phone">
          <span class="material-icons" style="font-size:14px;vertical-align:middle;">phone</span>
          ${order.customer_phone || 'N/A'}
        </span>
        <small class="customer-address">
          <span class="material-icons" style="font-size:14px;vertical-align:middle;">location_on</span>
          ${order.delivery_address || order.customer_default_address || 'No address'}
        </small>
      `;

      // Determine available actions based on status
      let actionsHtml = '';

      if (order.status === 'assigned') {
        actionsHtml = `
          <button class="btn-icon btn-success" title="Mark On Delivery"
                  onclick="markOnDelivery(${order.id})">
            <span class="material-icons">local_shipping</span>
          </button>
          <button class="btn-icon btn-danger" title="Cancel Delivery"
                  onclick="cancelDelivery(${order.id})">

            <span class="material-icons">check_circle</span>

          <button class="btn-icon btn-danger" title="Cancel Delivery"
                  onclick="cancelDelivery(${order.id})">
            <span class="material-icons">cancel</span>
          </button>
        `;

      tr.innerHTML = `
        <td>
          <input type="checkbox" class="custom-checkbox delivery-checkbox" data-order-id="${order.id}">
        </td>
        <td class="drag-handle-cell">
          <span class="material-icons drag-handle">drag_indicator</span>

  function initSortable() {
    destroySortable(); // Destroy any existing instance first

      handle: '.drag-handle',       // Only drag from the handle icon
      animation: 200,               // Smooth animation duration
      ghostClass: 'sortable-ghost', // Class on the drop placeholder
      chosenClass: 'sortable-chosen', // Class on the chosen item
      dragClass: 'sortable-drag',   // Class on the dragging item

        // After drag-and-drop, save new priority order

  function destroySortable() {
    if (sortableInstance) {
      sortableInstance.destroy();
      sortableInstance = null;

  function savePriorityOrder() {
    const rows = document.querySelectorAll('#deliveriesBody tr[data-order-id]');
    const priorities = [];

    rows.forEach((row, index) => {
      const orderId = parseInt(row.dataset.orderId);
      const newPriority = index + 1;
      priorities.push({ order_id: orderId, priority_order: newPriority });

      // Update the visual priority number
      const prioritySpan = row.querySelector('.priority-number');
      if (prioritySpan) prioritySpan.textContent = newPriority;
    });

    // Send new priority order to server
    ajaxPost('/api/riders/update_priority.php', {
      priorities: JSON.stringify(priorities)

      if (data.success) {
        showToast('Delivery priority updated', 'success');
      } else {
        showToast(data.message || 'Failed to update priority', 'error');
        loadDeliveries(); // Reload to revert
      }
    }).catch(err => {
      showToast('Failed to save priority order', 'error');
      loadDeliveries();
    });
  }

  function formatStatus(status) {
    const labels = {
      pending: 'Pending', confirmed: 'Confirmed', assigned: 'Assigned',
      on_delivery: 'On Delivery', delivered: 'Delivered', accepted: 'Accepted',
      ready_for_pickup: 'Ready for Pickup', picked_up: 'Picked Up', cancelled: 'Cancelled'
    };
    return labels[status] || status;

  function formatDate(dateStr) {
    if (!dateStr) return '—';
    const d = new Date(dateStr);
    return d.toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })
      + ' ' + d.toLocaleTimeString('en-US', { hour: '2-digit', minute: '2-digit' });
  }

  function escapeHtml(text) {
    const div = document.createElement('div');
    div.textContent = text;
    return div.innerHTML;
  }

  function renderPagination(total, page, perPage) {
    const totalPages = Math.ceil(total / perPage);
    const container = document.getElementById('deliveriesPagination');
    container.innerHTML = '';
    if (totalPages <= 1) return;

    const prevBtn = document.createElement('button');
    prevBtn.className = 'page-btn';
    prevBtn.textContent = 'Previous';
    prevBtn.disabled = page <= 1;
    prevBtn.addEventListener('click', () => { currentPage--; loadDeliveries(); });
    container.appendChild(prevBtn);

    for (let i = Math.max(1, page - 2); i <= Math.min(totalPages, page + 2); i++) {
      const pb = document.createElement('button');
      pb.className = `page-btn ${i === page ? 'active' : ''}`;
      pb.textContent = i;
      pb.addEventListener('click', () => { currentPage = i; loadDeliveries(); });
      container.appendChild(pb);
    }

    const nextBtn = document.createElement('button');
    nextBtn.className = 'page-btn';
    nextBtn.textContent = 'Next';
    nextBtn.disabled = page >= totalPages;
    nextBtn.addEventListener('click', () => { currentPage++; loadDeliveries(); });
    container.appendChild(nextBtn);

    const info = document.createElement('span');
    info.className = 'pagination-info';
    info.textContent = `Showing ${(page - 1) * perPage + 1}–${Math.min(page * perPage, total)} of ${total} deliveries`;
    container.appendChild(info);
  }

});
```

### `rider/js/delivery_history.js`

```javascript
document.addEventListener('DOMContentLoaded', function() {

  let currentFilter = 'all';
  let currentPage = 1;
  let searchText = '';
  let dateFrom = '';
  let dateTo = '';
  const perPage = 50;

  // Load stats and history on page load
  loadStats();
  loadHistory();

  // ===== PERFORMANCE STATS =====
  function loadStats() {
    // Fetch rider-specific stats
    ajaxGet('/api/riders/get_statistics.php', { self: true })
      .then(data => {
        document.getElementById('totalDelivered').textContent = data.completed || 0;
        document.getElementById('totalCancelled').textContent = data.cancelled || 0;
        const rate = data.completed > 0
          ? ((data.completed / (data.completed + data.cancelled)) * 100).toFixed(1)
          : '0.0';
        document.getElementById('successRate').textContent = rate + '%';
      })
      .catch(() => {
        document.getElementById('totalDelivered').textContent = '—';
        document.getElementById('totalCancelled').textContent = '—';
        document.getElementById('successRate').textContent = '—';
      });
  }

  // ===== FILTER BUTTONS =====
  document.querySelectorAll('#historyFilters .filter-btn').forEach(btn => {
    btn.addEventListener('click', function() {
      document.querySelectorAll('#historyFilters .filter-btn').forEach(b => b.classList.remove('active'));
      this.classList.add('active');
      currentFilter = this.dataset.filter;
      currentPage = 1;
      loadHistory();
    });
  });

  // ===== SEARCH (debounced) =====
  let searchTimeout;
  document.getElementById('historySearch').addEventListener('input', function() {
    clearTimeout(searchTimeout);
    searchTimeout = setTimeout(() => {
      searchText = this.value.trim();
      currentPage = 1;
      loadHistory();
    }, 300);
  });

  // ===== DATE RANGE =====
  document.getElementById('dateFrom').addEventListener('change', function() {
    dateFrom = this.value;
    currentPage = 1;
    loadHistory();
  });
  document.getElementById('dateTo').addEventListener('change', function() {
    dateTo = this.value;
    currentPage = 1;
    loadHistory();
  });

  // ===== LOAD HISTORY =====
  function loadHistory() {
    const statusFilter = currentFilter === 'all' ? 'delivered,cancelled' : currentFilter;

    const params = {
      status: statusFilter,
      search: searchText,
      page: currentPage,
      date_from: dateFrom,
      date_to: dateTo,
      scope: 'history' // Tell API to return completed/cancelled orders for rider
    };

    document.getElementById('historyBody').innerHTML =
      '<tr class="loading-row"><td colspan="9"><div class="spinner"></div></td></tr>';
    document.getElementById('emptyHistory').style.display = 'none';

    ajaxGet('/api/orders/get_orders.php', params)
      .then(data => {
        if (!data.orders || data.orders.length === 0) {
          document.getElementById('historyBody').innerHTML = '';
          document.getElementById('emptyHistory').style.display = 'flex';
          document.getElementById('historyPagination').innerHTML = '';
          return;
        }

        document.getElementById('emptyHistory').style.display = 'none';
        renderHistory(data.orders);
        renderPagination(data.total, data.page, data.per_page);
      })
      .catch(() => showToast('Failed to load delivery history', 'error'));
  }

  // ===== RENDER HISTORY TABLE =====
  function renderHistory(orders) {
    const tbody = document.getElementById('historyBody');
    tbody.innerHTML = '';

    const paymentLabels = { cod: 'Cash on Delivery', pickup: 'Pickup at Store', online: 'Online' };

    orders.forEach(order => {
      const tr = document.createElement('tr');

      const itemsHtml = order.items.map(item =>
        `<span class="item-with-icon">
          <img src="${item.item_icon || '/assets/images/default-item.png'}" class="item-icon-sm">
          ${item.item_name}
        </span>`
      ).join('<br>');

      const totalQty = order.items.reduce((sum, i) => sum + parseInt(i.quantity), 0);

      const customerDetails = `
        <strong>${order.customer_name || 'Unknown'}</strong>
        <span class="customer-phone">
          <span class="material-icons" style="font-size:14px;vertical-align:middle;">phone</span>
          ${order.customer_phone || 'N/A'}
        </span>
        <small class="customer-address">
          <span class="material-icons" style="font-size:14px;vertical-align:middle;">location_on</span>
          ${order.delivery_address || order.customer_default_address || 'No address'}
        </small>
      `;

      // For cancelled orders by rider, the cancellation reason is shown
      // Note: When rider cancels, the order reverts to Confirmed — it's not truly "cancelled"
      // But the history might show orders that were later cancelled by staff after rider returned them
      // The API should return the cancellation_reason if status is cancelled
      let cancelReasonHtml = '<span class="text-muted">—</span>';
      if (order.status === 'cancelled' && order.cancellation_reason) {
        cancelReasonHtml = `<span class="truncate-text" title="${escapeHtml(order.cancellation_reason)}">${escapeHtml(order.cancellation_reason)}</span>`;
      }

      tr.innerHTML = `
        <td><strong>#${order.id}</strong></td>
        <td class="customer-details-cell">${customerDetails}</td>
        <td class="items-cell">${itemsHtml}</td>
        <td>${totalQty}</td>
        <td>${paymentLabels[order.payment_type] || order.payment_type}</td>
        <td><span class="badge badge-${order.status.replace(/_/g, '-')}">${formatStatus(order.status)}</span></td>
        <td>${formatDate(order.order_date)}</td>
        <td>${order.delivered_at ? formatDate(order.delivered_at) : '<span class="text-muted">—</span>'}</td>
        <td class="cancellation-cell">${cancelReasonHtml}</td>
      `;

      tbody.appendChild(tr);
    });
  }

  // ===== HELPER FUNCTIONS =====
  function formatStatus(status) {
    const labels = {
      pending: 'Pending', confirmed: 'Confirmed', assigned: 'Assigned',
      on_delivery: 'On Delivery', delivered: 'Delivered', accepted: 'Accepted',
      ready_for_pickup: 'Ready for Pickup', picked_up: 'Picked Up', cancelled: 'Cancelled'
    };
    return labels[status] || status;
  }

  function formatDate(dateStr) {
    if (!dateStr) return '—';
    const d = new Date(dateStr);
    return d.toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })
      + ' ' + d.toLocaleTimeString('en-US', { hour: '2-digit', minute: '2-digit' });
  }

  function escapeHtml(text) {
    const div = document.createElement('div');
    div.textContent = text || '';
    return div.innerHTML;
  }

  function renderPagination(total, page, perPage) {
    const totalPages = Math.ceil(total / perPage);
    const container = document.getElementById('historyPagination');
    container.innerHTML = '';
    if (totalPages <= 1) return;

    const prevBtn = document.createElement('button');
    prevBtn.className = 'page-btn';
    prevBtn.textContent = 'Previous';
    prevBtn.disabled = page <= 1;
    prevBtn.addEventListener('click', () => { currentPage--; loadHistory(); });
    container.appendChild(prevBtn);

    for (let i = Math.max(1, page - 2); i <= Math.min(totalPages, page + 2); i++) {
      const pb = document.createElement('button');
      pb.className = `page-btn ${i === page ? 'active' : ''}`;
      pb.textContent = i;
      pb.addEventListener('click', () => { currentPage = i; loadHistory(); });
      container.appendChild(pb);
    }

    const nextBtn = document.createElement('button');
    nextBtn.className = 'page-btn';
    nextBtn.textContent = 'Next';
    nextBtn.disabled = page >= totalPages;
    nextBtn.addEventListener('click', () => { currentPage++; loadHistory(); });
    container.appendChild(nextBtn);

    const info = document.createElement('span');
    info.className = 'pagination-info';
    info.textContent = `Showing ${(page - 1) * perPage + 1}–${Math.min(page * perPage, total)} of ${total}`;
    container.appendChild(info);
  }

});
```

### `rider/js/settings.js`

```javascript
document.addEventListener('DOMContentLoaded', function() {

  // ===== PROFILE FORM =====
  document.getElementById('profileForm').addEventListener('submit', function(e) {
    e.preventDefault();

    const data = {
      full_name: document.getElementById('profileFullName').value.trim(),
      email: document.getElementById('profileEmail').value.trim(),
      phone: document.getElementById('profilePhone').value.trim()
    };

    if (!data.full_name || !data.email || !data.phone) {
      showToast('All fields are required', 'warning');
      return;
    }

    // Basic email validation
    if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(data.email)) {
      showToast('Please enter a valid email address', 'warning');
      return;
    }

    const submitBtn = this.querySelector('button[type="submit"]');
    submitBtn.disabled = true;
    submitBtn.innerHTML = '<div class="spinner" style="width:18px;height:18px;"></div> Saving...';

    ajaxPost('/api/settings/update_profile.php', data)
      .then(res => {
        if (res.success) {
          showToast('Profile updated successfully', 'success');
        } else {
          showToast(res.message || 'Failed to update profile', 'error');
        }
      })
      .catch(() => showToast('An error occurred', 'error'))
      .finally(() => {
        submitBtn.disabled = false;
        submitBtn.innerHTML = '<span class="material-icons">save</span> Save Changes';
      });
  });

  // ===== PASSWORD FORM =====
  document.getElementById('passwordForm').addEventListener('submit', function(e) {
    e.preventDefault();

    const current = document.getElementById('currentPassword').value;
    const newPass = document.getElementById('newPassword').value;
    const confirm = document.getElementById('confirmPassword').value;

    if (!current || !newPass || !confirm) {
      showToast('All password fields are required', 'warning');
      return;
    }
    if (newPass.length < 6) {
      showToast('New password must be at least 6 characters', 'warning');
      return;
    }
    if (newPass !== confirm) {
      showToast('New passwords do not match', 'warning');
      return;
    }

    const submitBtn = this.querySelector('button[type="submit"]');
    submitBtn.disabled = true;
    submitBtn.innerHTML = '<div class="spinner" style="width:18px;height:18px;"></div> Changing...';

    ajaxPost('/api/settings/change_password.php', {
      current_password: current,
      new_password: newPass,
      confirm_password: confirm
    }).then(res => {
      if (res.success) {
        showToast('Password changed successfully', 'success');
        this.reset();
      } else {
        showToast(res.message || 'Failed to change password', 'error');
      }
    })
    .catch(() => showToast('An error occurred', 'error'))
    .finally(() => {
      submitBtn.disabled = false;
      submitBtn.innerHTML = '<span class="material-icons">lock_reset</span> Change Password';
    });
  });

  // ===== AVAILABILITY TOGGLE (in Settings) =====
  const settingsToggle = document.getElementById('settingsAvailToggle');
  if (settingsToggle && !settingsToggle.disabled) {
    settingsToggle.addEventListener('change', function() {
      const newState = this.checked;
      const confirmMsg = newState
        ? 'Set yourself as Available? You may receive new delivery assignments.'
        : 'Set yourself as Unavailable? You will not receive new delivery assignments.';

      Swal.fire({
        title: newState ? 'Go Available?' : 'Go Unavailable?',
        text: confirmMsg,
        icon: 'question',
        showCancelButton: true,
        confirmButtonText: newState ? 'Yes, Go Available' : 'Yes, Go Unavailable',
        cancelButtonText: 'Cancel',
        confirmButtonColor: newState ? 'var(--success)' : 'var(--warning)',
      }).then(result => {
        if (result.isConfirmed) {
          ajaxPost('/api/riders/toggle_availability.php', {})
            .then(data => {
              if (data.success) {
                showToast(
                  newState ? 'You are now available for deliveries' : 'You are now unavailable',
                  'success'
                );
                // Update the status text
                const statusText = document.getElementById('settingsAvailStatus');
                statusText.textContent = newState ? 'Available' : 'Unavailable';
                statusText.className = newState ? 'text-success' : 'text-danger';
              } else {
                settingsToggle.checked = !newState;
                showToast(data.message || 'Failed to update availability', 'error');
              }
            })
            .catch(() => {
              settingsToggle.checked = !newState;
              showToast('An error occurred', 'error');
            });
        } else {
          settingsToggle.checked = !newState;
        }
      });
    });
  }

});
```

## IMPORTANT IMPLEMENTATION NOTES FOR RIDER PAGES

1. **Rider accounts are always `status='active'`** from creation. There is no pending or flagged state for riders. No need to check for pending/flagged account status on rider pages (unlike customer pages). However, always verify the session role is `ROLE_RIDER` on every page.

2. **Riders ONLY see orders assigned to them.** The `api/orders/get_orders.php` endpoint scopes results by `rider_id = session.user_id` when the role is `rider`. The rider frontend should never display or attempt to access orders not assigned to them.

3. **Customer Details format in tables** must always be displayed as:
   ```
   Customer Full Name
   📞 Phone Number
   📍 Delivery Address (from the order's delivery_address field, or customer's default address)
   ```
   Use Material Icons (`phone`, `location_on`) inline before the phone and address. Each on its own line within the cell.

4. **Drag-and-drop priority reordering** uses Sortable.js attached to the `<tbody>` element. The `.drag-handle` class on the grip icon makes only that icon draggable (not the entire row). After each drag-and-drop, the new order is saved immediately to the server via `api/riders/update_priority.php`. Visual priority numbers are updated in real-time. The ghost/placeholder row during dragging should have reduced opacity and a subtle primary color background.

5. **Availability toggle appears in TWO places:** the Dashboard and the Settings page. Both use the same API endpoint (`api/riders/toggle_availability.php`). Both must show the confirmation SweetAlert before toggling. Both must handle the disabled/locked state when the rider has on_delivery orders. The toggle is a visual `<input type="checkbox">` styled as a toggle switch using the `.toggle-switch` and `.toggle-slider` CSS classes from `components.css`.

6. **When a rider cancels a delivery**, the order does NOT get status `cancelled`. Instead, it reverts to `confirmed` with `rider_id` set to NULL. This is a key distinction — the rider is just declining/returning the delivery, not cancelling the customer's order. Staff/Admin will then reassign to another rider. The cancellation reason from the rider is stored for Staff/Admin reference. The frontend dialog should say "Cancel Delivery" and explain "The order will be sent back to staff for reassignment."

7. **Performance stats on delivery_history.php** show:
   - Total Delivered: Count of orders with status `delivered` (or `accepted` or `picked_up`) WHERE `rider_id = this rider`
   - Total Cancelled: Count of orders that were reassigned away from this rider (this is tracked differently — the API should count based on session logs or a rider_cancelled flag. Alternatively, simplify: count orders in history where status = cancelled AND cancelled_by = rider's user_id)
   - Success Rate: `Completed / (Completed + Cancelled) × 100`%
   - The `api/riders/get_statistics.php` with `self=true` parameter returns these stats for the logged-in rider.

8. **Item icons** in tables: same as customer — if item has `item_icon`, show that image. If NULL, show `/assets/images/default-item.png`. Use `.item-icon-sm` class (20-24px, rounded corners).

9. **Every AJAX call** must use `ajaxPost()` or `ajaxGet()` from `global.js` which automatically attach the CSRF token. Every call must have `.catch()` error handling that shows a toast.

10. **All dates** must be formatted in Philippine timezone. Use the JS `formatDate()` function which renders dates as "Jun 30, 2025 02:30 PM".

11. **Each page-specific JS file** must be wrapped in `document.addEventListener('DOMContentLoaded', function() { ... })`.

12. **Dark mode support** is handled entirely via CSS variables. All page-specific CSS should ONLY use CSS custom properties for colors — never hardcode colors. This ensures dark mode works automatically when `data-theme="dark"` is set on `<html>`.

13. **The rider has NO access to:** placing orders, viewing inventory, managing accounts, viewing analytics, session logs, or system settings. The sidebar only shows: Dashboard, Assigned Deliveries, Delivery History, Settings. If a rider tries to access any other page, the `require_role([ROLE_RIDER])` check and proper URL routing should prevent it.

14. **Tables render max 50 items per page** with pagination showing Previous/Next buttons and "Showing X-Y of Z items" counter.

15. **Glass card effect** is used for: stat cards, settings sections, availability toggle card on dashboard. Not overused — keep it subtle and professional.

16. **Bulk actions bar** appears only when one or more checkboxes are checked. It slides down with animation. When no items are checked, it hides. The "Select All" checkbox in the table header toggles all visible checkboxes. Bulk actions respect the same rules as individual actions — e.g., bulk "Mark Delivered" only applies to `on_delivery` orders, bulk "Mark On Delivery" only applies to `assigned` orders. If a mixed selection is made, the API processes valid ones and reports how many succeeded vs failed.
